January 2013 has seen a severe error in the forum text editor, removing numerous user-friendly editing features. However, the text can still be modified by use of BBCodes. For the convenience of present and future forum members, here is a comprehensive list of all BBCodes currently known to work with the 910CMX forum.

If a BBCode is not listed, please post it in the comments, and it will be added to this list. Please preview the post before submitting, to be sure that the BBCode format works.

WARNING! Currently, posting the less-than sign (<>) without previewing the post will successfully post said sign. Previewing the post WITH the less-than sign results the text and signs displaying in the preview, but in the editor, everything after the less-than sign being deleted, until the next greater-than sign (>). Which means, if you aren't careful with the use of the less-than sign, your post could end up accidentally deleted. Until this is fixed, DO NOT USE THE LESS-THAN SIGN.

 [hr] Horizontal Line

[i]italics[/i]

[b]bold[/b]

[u]underline[/u]

[s]strikethrough[/s]

[sup]superscript (and yes, they stack)[/sup]

[sub]subscript(these stack toooooooo)[/sub]

[color=#RRGGBB]color change (color values are in hexadecimal, from 00 to FF)[/color]
Replacing the hexadecimal code with a preset color also works. Confirmed examples:
  • [color=red]Red[/color]
  • [color=orange]Orange[/color]
  • [color=yellow]Yellow[/color]
  • [color=green]Green[/color]
  • [color=blue]Blue[/color]
  • [color=purple]WALUIGI TIME![/color]
  • [color=gray]Gray[/color]
  • [color=white]Spoilers[/color]
  • [color=black]Black (font is this by default)[/color]
  • Color names that do NOT work (hexadecimal must be used): light_gray, magenta, pink, brown, dark_green, cyan, dark_blue
[size=#]resized text. Replace # with 1 2 3 4 5 6 7 8; text is automatically at size=4)[/size]

[font=font name]Font Change[/font]

[acronym="text here"]other text here[/acronym] Text with mouseover text.
(WARNING! Using [acronym] tags, followed by tags is CONFIRMED to hide ALL content within the [acronym] tags, and all content following the [acronym] tags until the next [[url] tag, in the submitted post. The hidden content code is visible in the text editor.)

[url="URL here"]text here hyperlinked text
[url]URL here[/url] automatically reverts to the above format as: [url="URL here"]URL here[/url]
(WARNING! The URL will only work if it begins with "http://"; else, it will not work at all.

[img]Image URL here[/img]
Image. To find the Image URL, right-click the image, click "Copy Image URL", and paste where indicated. Note that it must be on an image-hosting website. The 910CMX gallery is recommended but not required, but other hosting sites have caused problems on their end.
(WARNING! Previewing an image will display the image properly, but will alter the text in the display. As the Image url is a URL, it will convert it from [img]Image URL[/img] to [img][url="Image URL"]Image URL[/url][/img]. Previewing THAT will display a hyperlinked url, and will alter the text in the editor to convert the image URL into [url] tags a second time.

[media]Media URL here[/media]
Embedded media. For instance, a YouTube URL placed in [media] tags will display the YouTube video player itself, instead of a hyperlink to the page.

[quote]Quoted Text[/quote] Note: hitting the ENTER key after the [quote] tag results in an unnecessary whitespace within the quote, as below:



(seriously, that whitespace can be avoided)
[quote name="name"]Quote attributed to a person (Note: currently, the name will not display)[/quote]

Clicking the "quote" button on someone's post will return the format: [quote name="name" post="number" timestamp="bigger number"][/quote]. This displays when the original post was made, and by who. As with the "name" stamp, this is currently non-functional.
[spoiler]Spoilered Text[/spoiler]
Spoiler
The Game

[list=#]
[*]Item 1
[*]Item 2
[*]etc...
[/list]
  • List format
  • If #=0, or greater than one (as [list=2]), or not specified (as [list]), it will be a list with bullet points.
  • If #=1, it will be a numbered list.
  • Please follow the ending tag [/list] with a non-whitespace key. The reason is because if the ending tag is used, followed by the Enter key (or by whitespace and THEN the Enter key), it will delete that Enter key in the text editor and cause unintended alterations to the spacing.
[code=auto:0]C++ code[/code] (note that [code][/code] automatically converts to [code=auto:0][/code])
(WARNING! C++ code may make use of the less-than symbol. See the note at the top for why this is important.)

Some things to add.

  • Spoilers work now! Yay!
  • The rich text editor works, with a few caveats. You can only use it if you go to preview the post. Toggling between the editors deletes your post. And sometimes the post will be broken when posted; this can be fixed by editing, previewing, and saving. (This info comes from King of Ferrets)

Let me do some testing.
Spoiler
&lt;test>
Test 1: URL AFTER acronym [acronym][/acronym][url][/url]
If you hit "quote", you'll see that the part of this paragraph that starts at the acronymed word "paragraph" is completely deleted, right up until the URL. As a distraction and filler, I have to point out that the secret in my post over in the Last Post Wins thread is the number of words spoken by each character per line. Go take a look.

Test 2: URL WITHIN acronym [acronym][url][/url][/acronym]
Gunnerkrigg Court link.

Test 3: Image WITHIN acronym [acronym][img]image[/img][/acronym]


Test 4: Image AFTER acronym [acronym][/acronym][img]image[/img]


Test 5: Following all that with a url
URL deleted post-testing for visibility purposes
&lt;/test>

And the results:
Spoiler
  • Using [acronym] tags, followed by [img] tags presents no problem in of itself. Whether this was recently fixed or never a problem at all is unknown.
  • Using [acronym] tags, followed by [url] tags causes everything from the start of the [acronym] tags to the start of the [url] tags to be erased, as in this example with strikethrough representing deleted text: The quick [acronym="Yes, foxes are red. Do we care?"]brown fox[/acronym] jumps over the lazy [url="http://en.wikipedia.org/wiki/Dog"]dog[/url].
  • A possible solution to this problem: Present an empty [acronym] before the [url], like this: [acronym=""].[/acronym][url="page url"]text[/url] This solution will not work under the conditions of Test 2.
  • Test 5 deleted everything after the starting [acronym] tag for Test 2, ending with the opening tag within Test 5. Reinforcing the notion that using [acronym], followed by [img], followed by [url] is not recommended.
  • A repeat of test 5, with the url outside of the spoiler box, caused the spoiler tags to break, resulting in even worse errors.
  • If the URL includes "http://" at the beginning, it will cause the errors with the [acronym] tags.
  • If the URL does NOT include "http://", the URL tags will fail, resulting in plain, unhyperlinked text with no tags.
  • Previewing an image will display the image properly, but will alter the text in the display. As the Image url is a URL, it will convert it from [img]Image URL[/img] to [img][url="Image URL"]Image URL[/img]. Previewing THAT will display a hyperlinked url, and will alter the text in the editor to convert the image URL into [url] tags a second time.
  • &lt; converts to &lt; in the preview and displays as < in the editor. Previewing/posting THAT will display the < symbol in the preview/post and delete text in the editor. It only seems to work in an attempt to make html tags.
